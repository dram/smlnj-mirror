10.1 Time stamps

For skeleton files and binfiles, CM uses file system time stamps (i.e., modification time) to determine whether a file has become outdated. The rule is that in order to be considered ``up-to-date'' the time stamp on skeleton file and binfile has to be exactly the same9 as the one on the ML source file. This guarantees that all changes to a source will be noticed--even those that revert to an older version of a source file.10

CM also uses time stamps to decide whether tools such as ML-Yacc or ML-Lex need to be run (see Section 6). However, the difference is that a file is considered outdated if it is older than its source. Some care on the programmers side is necessary since this scheme does not allow CM to detect the situation where a source file gets replaced by an older version of itself.
