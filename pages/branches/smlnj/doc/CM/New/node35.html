4.3.1 Sharing annotations

ML source files in CM description files can be specified as being private or shared. This is done by adding a tool parameter specification for the file in the library- or group description file (see Section 6). To mark an ML file as private, follow the file name with the word private in parentheses. For shared ML files, replace private with shared.

An ML source file that is not annotated will typically be treated as shared unless it statically depends on some other private source. It is an error, checked by CM, for a shared source to depend on a private source.
