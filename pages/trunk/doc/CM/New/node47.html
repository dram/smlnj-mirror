6.2.2 ML-Lex

The ML-Lex tool governs files that are input to the ML-Lex lexical analyzer generator [AMT89]. Its class name is mllex. Recognized file name suffixes are .lex and .l. For a source file , the tool produces one targets .sml which will always be treated as ML source code. Tool parameters are passed on without change to that file.

The tool invokes the ml-lex command if the target is outdated (just like in the case of ML-Yacc). Unless anchored using the path anchor mechanism (see Section 3.3), the command ml-lex will be located using the operating system's path search mechanism (e.g., the $PATH environment variable).
