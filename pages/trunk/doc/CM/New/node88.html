16. Some history

Although its programming model is more general, CM's implementation is closely tied to the Standard ML programming language [MTHM97] and its SML/NJ implementation [AM91].

The current version is preceded by several other compilation managers. Of those, the most recent went by the same name ``CM'' [Blu95], while earlier ones were known as IRM ( Incremental Recompilation Manager) [HLPR94b] and SC (for Separate Compilation) [HLPR94a]. CM owes many ideas to SC and IRM.

Separate compilation in the SML/NJ system heavily relies on mechanisms for converting static environments (i.e., the compiler's symbol tables) into linear byte stream suitable for storage on disks [AM94]. However, unlike all its predecessors, the current implementation of CM is integrated into the main compiler and no longer relies on the Visible Compiler interface.
