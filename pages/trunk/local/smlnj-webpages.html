The old CVS-based content is still available in a sub-directory named old-site-from-cvs. We should remove this once we have confirmed that everything works ok. Once this is the case, we should do the same with the "main" site (currently hosted by SourceForge) to avoid consistency problems.
If you want to edit the smlnj pages, you should check out your own working copy of the pages/trunk directory, make your edits in your working copy, and then commit them to the svn repository.

Your changes can can be made visible at the main UofC smlnj site (also available at URL http://smlnj.cs.uchicago.edu) by doing an svn update in the htdocs directory mentioned above.
